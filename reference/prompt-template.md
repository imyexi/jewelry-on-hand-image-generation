# 真人参考底图首饰替换 Prompt 模板

`prompt_builder.build_generation_prompt()` 是唯一生产入口。Prompt 使用已确认参考构图快照、产品 analysis 与 canonical 构建；不得手写自由 Prompt 绕过 validator。

## 固定编辑开头

第一段必须表达：

```text
这是参考底图编辑任务。以参考底图为底图进行编辑，保持未授权区域的画面结构。
只允许移除参考图原首饰并在同一位置换入一件目标产品，
以及重建必要接触阴影、局部遮挡和受力关系；可清除小面积水印或平台标识。
```

参考底图是人物、姿势、手势、构图、景别、服装、背景、光线、留白和替换位置的唯一来源。产品上手图只提供目标珠宝身份。任何产品规则、输出规格或风格词都不能出现在该声明之前。

## 内部图职责

### 内部图 1：参考底图

只从确认快照读取并锁定：人物身份与可见身体区域、身体姿势、手势、手臂角度、掌向、手指关系、服装、发型、背景、道具、镜头角度、景别、裁切、主体位置、留白、光线方向、明暗、色温、整体色调和唯一替换位置。

参考底图中的全部原首饰都属于清除对象，不提供目标产品身份。

### 内部图 2：产品身份图

只提供：品类、数量、珠子/链条/戒圈/主石/吊坠/金属件、排列、连接、方向、比例、颜色、材质、透明度、纹理、反光和可见尺寸感。

若内部图 2 是多张真人产品上手图确定性拼接成的同一件产品多视角身份图，各分区只表示同一件产品的不同可见面，不表示多件、双圈或重复生成。不得使用 AI 修改产品像素，不得使用白底或平铺图补齐视角；画面仍只能出现一件目标产品。

禁止迁移内部图 2 中的人物、手腕、手臂、手、手指、颈部、胸部、衣服、头发、脸、皮肤块、肤色或背景。产品细节图只用于 review、结构分析和 QC，不作为模型第三张输入。

## 允许与禁止修改

只允许：

1. 移除参考底图原首饰和直接接触阴影；
2. 在快照确认的同一位置放入一件目标产品；
3. 重建新产品必要接触、遮挡、受力和局部阴影；
4. 处理小面积水印、logo 或平台标识。

禁止：重新生成人物或环境、换脸、换肤色、换姿势、换手势、换衣、换景、改变人物位置、改变镜头、裁切、推进、放大、缩小或把生活场景改成产品特写。展示面积不足时换参考图，不在生成阶段补救。

## 快照字段拼装

确认快照优先于所有动态规则。按顺序写入：

- `output_role`、`framing`、`camera_angle`、`subject_placement`；
- `visible_body_regions`、`pose.body|arm|hand|hand_side`；
- `clothing`、`background`、`lighting`；
- `replacement_target`、`other_jewelry_to_remove`、`text_or_ui_risk`。

产品 analysis、角色模板或品类策略与上述任一字段冲突时停止，要求新建 run 并重新 `prepare-review`。不能在 Prompt 中折中或覆盖快照。

## 产品 canonical

canonical 只控制珠宝身份与佩戴物理：

- `must_keep` 的名称、问题和结构语义全部保留；
- 颜色、材质、透明度、纹理、反光、数量、排列、连接和比例不得美化改款；
- 项链使用 `schema_version=2` 与 `pendant_semantics`；普通项链不新增主吊坠，带链吊坠保留恰好一个主吊坠及所属层和连接；
- 不推断不可见扣头、背面、连接或戒圈背面，不自动补链。

## 品类策略

### `bracelet`

保持珠序、主珠、配珠、隔圈与金属件，自然环绕快照指定手腕。不得复制产品或迁移产品图手腕。

### `necklace` / `pendant_necklace`

保持同一产品 1 至 3 层、长度等级、层间落差、主吊坠数量/所属层/方向/连接。`worn` 或 `hand_held` 只影响佩戴物理，不注入“锁骨近景”“半身”等构图。

### `ring`

内部图2是戒指身份唯一来源，固定 `ring_count=1`、`hand_side`、`finger_position`、`ring_wear_style=finger_base`。戒指身份唯一来源仅决定戒圈、戒面/主石、镶嵌、颜色和材质。最高优先级：只生成一枚目标戒指；禁止生成手镯、手链、第二枚戒指。按快照保留手背/掌侧朝向和手指姿势，让戒圈自然环绕手指；不可见戒圈背面不推断。

## 角色策略

- `hand_worn`：保留手侧、掌向、手臂角度、手指姿势、目标部位和景别；项链 `hand_held` 也必须服从快照。
- `lifestyle`：保留半身、全身或环境景别、人物位置、服装、道具和生活环境；产品较小也不得裁成特写。
- `hero`：当前 builder 必须拒绝并指向独立主图 Skill。

## 固定层序

1. 底图编辑声明；
2. 双图职责；
3. 确认快照；
4. 允许修改区域；
5. 产品 canonical；
6. 品类佩戴物理；
7. 角色保留要求；
8. 通用禁止项与输出质量。

## 预算与验证

Prompt 不超过 1200 字。超预算时删除重复修饰，不得删除底图锁定、两图职责、快照字段、允许修改范围、canonical 或冲突停止。

以 `scripts/validate_prompt_contract.py` 对 Prompt、确认快照、`product-analysis.json` 和 `product-fidelity-constraints.json` 做四输入验证。UTF-8、角色、摘要、字段或构图冲突必须在 provider 调用前返回非零。
