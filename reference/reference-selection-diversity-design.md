# 上手参考图多样性匹配优化设计

## 背景

当前参考图匹配以基础分排序后直接取 Top 3。实际跑批中，多个 SKU 会反复命中同一组参考图，例如暗调黑衣近景组 `1（30）/1（31）/1（32）`，或清透白衬衫组 `1 (1)/1 (10)/1 (11)`。这些图分数相同且在分类表中连续，排序用 `(-score, row.index)` 时会稳定取连续前三张，导致 AI 成图构图固定，缺少批量多样性。

## 目标

- Top 3 不再只是最高分前三张，而是“质量合格且互相差异明显”的三种参考方向。
- 保留 Rank 1 的安全保真能力，同时让 Rank 2、Rank 3 提供光线、场景、姿势或构图变化。
- 避免同一拍摄组、同一风格簇、同一场景簇在单个产品 Top 3 中重复。
- 批量处理多个 SKU 时，避免不同 SKU 反复使用同一张参考图；单 SKU 多样性不等于批次多样性。
- 修正过宽的“自然”风格匹配，避免所有产品被通用目标文案吸到清透自然组。
- 大珠产品不再只被近景规则锁死，而是优先看展示空间、裁切风险和手腕清晰度。

## 设计

### 候选生成

先保留现有硬过滤和基础评分，确保候选图质量不下降。基础排序仍按分数高低输出完整 `reference_candidates.json`，用于追踪和排查。

### 多样性重排

在基础排序后新增 diversity rerank：

1. 从基础排序中取质量池，默认覆盖全部候选；选择 Top 3 时优先考虑 `score >= max_score - 40` 的候选。
2. 第一张选择最高分候选，作为安全保真参考。
3. 后续候选按“基础分 - 重复惩罚 + 槽位奖励”计算多样性分。
4. 若候选与已选项共享同一风格簇、场景簇、姿势簇或拍摄组，则扣分。
5. 只要候选仍在质量阈值内，优先选择与已选项差异更大的图。

### 批次级多样性

当一次处理多个 SKU 时，不能把每个 SKU 当成独立任务从零开始选择参考图。批量流程应在单 SKU 多样性之外维护批次用量表：

1. 逐个 SKU 读取 `reference_candidates.json`，保留每个 SKU 的基础候选分和质量窗口。
2. 选择当前 SKU 的 Top 3 时，对本批次已使用过的 `source_file_name` 施加强惩罚，候选充足时不得复用同一张参考图。
3. 对本批次已大量使用的 `shoot_group` 和 `style_cluster` 施加较轻惩罚，避免整批都落在同一拍摄组或同一风格。
4. 批次级重排后必须重写对应 run 的 `analysis/selected_references.json`，并重新复制 `review/rank-N-*` 与生成 `review.html`。
5. 若候选池不足以完全去重，允许复用，但必须在汇总中标记复用原因和复用次数。

### 簇定义

- `style_cluster`：优先使用 `风格分类`；为空时按关键词归类为暗调、清透、户外、车内、对镜、其他。
- `scene_cluster`：按 `场景关键词` 和 `备注` 归类，例如暗光、白衬衫、户外、床品、车内、对镜、其他。
- `pose_cluster`：按 `推荐使用方式`、`用途分类`、`备注` 归类，例如手背、掌心、前臂、对镜、双手、手掌托物、近景。
- `shoot_group`：按文件名识别同一拍摄组；连续括号编号或同一微信图片前缀视为同组。

### 槽位语义

- Rank 1：安全保真，优先最高基础分、手腕清楚、风险低。
- Rank 2：生活方式，优先和 Rank 1 不同的光线/背景/场景。
- Rank 3：差异化构图，优先和 Rank 1/2 不同的姿势/构图。

### 评分修正

- `自然` 不再单独触发清透自然风格匹配；必须命中 `清透`、`自然光`、`白衬衫`、`浅色`、`奶油`、`柔和生活感` 等更具体词。
- 大珠产品将 `近景` 加分降低，新增“展示空间足/无裁切/手腕清晰”的概念沿用现有手腕露出和裁切风险信号。

## 验收标准

- 同一产品的 Top 3 不应全部来自同一 `风格分类` 或同一拍摄组，除非候选池本身不足。
- 分数相同且文件名连续的一组图，不应被直接取满 Top 3。
- 同一批次内，如果候选池充足，同一张参考图不应跨 SKU 重复进入 Top 3。
- `自然真实的小红书上手试戴图` 这类通用产品文案不应单独触发清透自然加分。
- 大珠产品仍能拿到近景安全参考，但 Top 3 中应允许生活方式或差异化构图进入。
- 现有硬过滤、放宽策略、风险记录、rank 连续性保持兼容。
