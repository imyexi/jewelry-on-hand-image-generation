# QC 检查清单

QC 结果只能使用三种状态：

- `pass`：成图满足所有核心要求，可交付或进入后续打包。
- `rerun`：方向可用，但存在可通过重新生成、调整参考图或强化提示词修复的问题。
- `reject`：存在严重质量、合规或产品错误，不应继续使用。

不得使用 `pending`、`ok`、`fail` 等其他状态。

## 必查项

每次 QC 必须逐项检查并记录通过项与失败项：

1. 产品保真：主珠、配珠、吊坠、金属件、颜色、透明度、纹理、反光和比例应忠实于产品图，不得缺失核心配件。
2. 关键识别点：必须读取 `analysis/product_fidelity_constraints.json`，逐项回答每个 `must_keep[].qc_question`。
3. 构图迁移：应迁移选定参考图的手势、佩戴位置、画幅关系和整体风格，但不能复制参考图中原有首饰。
4. 佩戴自然度：首饰应贴合手部或手腕，遮挡关系合理，不漂浮、不穿模、不像后期硬贴。
5. 手部真实性：手指数量、关节、皮肤纹理、指甲和手腕结构应自然可信，不能有明显畸形。
6. 手腕来源一致性：手腕宽度、手臂轮廓、肤色、皮肤纹理和上下肢体连接应连续自然，不能把产品原图中的粗手腕、局部手臂或皮肤块整体迁移到参考图手部上。
7. 禁止项：画面中不得出现文字、水印、logo、继承参考图首饰、核心配件缺失、明显手部畸形、源图手臂局部贴片。

## 关键识别点判定

`must_keep` 中任一关键识别点失败时，本次结果不得标记为 `pass`。

- `pass`：所有 `must_keep` 都保留，位置、形态和相邻关系可被肉眼识别。
- `rerun`：关键识别点轻微变形但仍可辨认，且画面整体方向值得重跑修复。
- `reject`：关键识别点缺失、被改款，或被泛化成普通圆珠、普通隔珠、普通链条、普通金属片等结构。

常见重点检查结构：

- 随形 / 随行：是否仍是不规则异形珠或切面件，而不是圆珠、椭圆珠。
- 跑环：是否仍是环状连接或活动环，而不是普通链坠、圆珠或金属片。
- 双尖：两端尖形轮廓是否保留。
- 回纹、雕刻、雕花：表面纹样是否保留，不能变成光面。
- 貔貅、桶珠、吊坠、流苏、链坠：造型、方向、连接关系是否保留。

## 禁止项判定

出现以下任一情况时，应记录到 `failed`：

- 文字：画面中出现非产品本身必要的文字、乱码、标签字样或说明文字。
- 水印：画面中出现平台水印、品牌水印、生成器水印或半透明标记。
- logo：画面中出现不属于产品的品牌 logo、平台 logo 或装饰性标识。
- 继承参考图首饰：参考图里的戒指、手链、手串、项链等被保留或混入成图。
- 源图手臂局部贴片：内部图2里的皮肤、粗手腕、手臂轮廓、掌纹、指甲或肤色块被和手串一并迁移到成图中。
- 核心配件缺失：主珠、明显吊坠、关键金属隔珠、核心装饰件等产品识别点缺失。
- 关键识别点失败：`must_keep` 中的结构缺失、变形或被泛化。
- 明显手部畸形：多指、断指、融指、关节错位、手掌扭曲、皮肤结构异常等一眼可见问题。

## QC JSON

`qc.json` 至少包含：

```json
{
  "status": "rerun",
  "passed": ["构图迁移正确", "无水印"],
  "failed": ["白水晶随形被改成圆珠"],
  "fidelity_checks": [
    {
      "name": "白水晶随形",
      "question": "白水晶随形是否仍是不规则透明异形珠",
      "result": "fail",
      "notes": "成图变成普通透明圆珠"
    }
  ],
  "notes": "关键识别点失败，不允许 pass"
}
```

`fidelity_checks[].result` 只能是 `pass`、`rerun` 或 `fail`。当任一结果不是 `pass` 时，整体 `status` 不能为 `pass`。

## `rerun` 与 `reject` 建议

- 使用 `rerun`：产品主体基本正确，构图方向可用，但存在轻微裁切、局部变形、佩戴不够自然、细节偏色、背景不理想等可通过再次生成修正的问题。
- 使用 `reject`：产品类型错误、核心配件缺失、参考图首饰被明显继承、出现水印或 logo、手部严重畸形、画面不可作为商品图使用，或多项关键要求同时失败。
