# 用户提示词约束参考图选择实施报告

## 实施范围

本次只执行 `reference/superpowers/specs/2026-07-16-用户提示词约束参考图选择设计.md`，修改通用 `jewelry-on-hand-workflow` 的参考图选择、审计绑定、生成前 gate、Prompt 隔离、测试和对应通用文档。未修改每周四上新文档，也未修改独立的 `skills/jewelry-product-hero-workflow/`。

## 最终行为

- `prepare-review` 新增可选参数 `--reference-selection-prompt`。系统只按中英文逗号、分号和换行拆分条件，去除空白和重复项，空格不拆词。
- 有提示词时，全部规范化条件都是硬约束。每张候选必须在白名单语义字段中命中全部条件；系统不自动放宽。
- 无提示词时，只按适用品类、输出角色和参考图关键词与产品需求的贴合度选择，不附加背景明暗或其他系统级风格默认值。
- 任一模式下不足 3 张合格候选都阻断。错误报告基础 gate 前后数量、逐条件命中数量和全部条件同时命中数量。
- 正式选择接口强制生成恰好三个互异 rank 的 Top 3；旧的两返回值包装器继续保留兼容。

## 审计与摘要绑定

每个 `prepare-review` run 写入 `analysis/reference_selection_constraints.json`，包括：

- `schema_version`、选择模式、原始提示词和规范化条件；
- 基础 gate 前、基础 gate 后、提示词 gate 后的候选数量；
- 逐条件命中数量、命中字段和逐候选证据；
- 基于稳定 JSON 编码计算的 SHA-256 摘要。

候选、`analysis/selected_references.json` Top 3 和生成类 `review/review_decision.json` 都写入相同的 `reference_selection_constraints_sha256`。`record-decision` 和 `generate` 会重新计算审计摘要，并校验 Top 3 恰好包含 rank 1、2、3 且每条摘要一致。审计、Top 3 或决策任一处被修改，都会在创建 generation 目录和调用 provider 前阻断。`rerank-batch` 只沿用现有审计；缺少审计的历史 run 不能重排。

## Prompt 隔离

Prompt builder 不接收选图提示词、规范化条件或候选命中证据。选图提示词不得写入 AIReiter Prompt。公共输出角色和戒指重试压缩文案已删除固定背景明暗要求，便携 Prompt 校验器也不再要求“深色背景”。

已选参考图自身的 `scene_keywords`、`pose_keywords`、风格和背景描述仍属于参考图元数据，可按原有职责进入 `【参考构图场景】`；这不等于把用户的选图提示词复制进最终 Prompt。

## 已知限制

- 选图条件使用白名单字段的大小写不敏感子串匹配，不做同义词扩展、分词或语义向量召回；这是为了让硬约束结果可审计、可复现。
- 历史 run 缺少 `analysis/reference_selection_constraints.json` 时保持只读，不能直接进入新的重排、决策或生成流程，必须重新执行 `prepare-review`。
- 全量测试中的历史 `output/` 依赖未复制到隔离 worktree；本报告保留完整失败日志并以聚焦回归作为本次功能验证的主要证据。
