## 2. 目标与非目标

### 2.1 目标

1. 将“跑环”从宽泛的环状结构描述升级为可生成、可检查的结构化约束。
2. 修正 QY002 的产品事实，使跑环进入 canonical `must_keep` 和逐项 QC。
3. 按已确认的人工反馈修订其余 6 个商品的本批次 canonical 约束。
4. 在不覆盖历史产物的前提下生成 11 张纠错图。
5. 仅将通过严格 QC 的图片加水印并替换飞书错误附件。
6. 回读验证 7 条记录均恰好包含 2 张、合计 14 张正确附件。

### 2.2 非目标

1. 不为 QY002 或其他 PN 增加商品编号专属代码分支。
2. 不把所有“小珠结”“珠结”自动识别成跑环。
3. 不把所有灰棕色木珠统一强制为深色。
4. 不在本次建立完整的环形珠序拓扑数据模型。
5. 不重跑未被人工判错的其他商品或其他视角。
6. 不覆盖、删除或伪装历史生成产物和历史评估记录。

## 3. 方案选择

采用“通用跑环规则增强 + 本批次定向约束修订”。

未采用仅在最终提示词追加自然语言的方案，因为该做法不能同步增强 canonical 约束和 QC，后续仍会漏检。未采用完整环形拓扑模型，因为它需要扩展分析 schema、人工确认界面和 QC 模型，超出本次纠错范围。

## 4. 跑环通用规则

### 4.1 触发边界

只有最终产品分析或人工确认明确出现“跑环”时，才生成跑环约束。“红色小珠结”“珠结”“金属连接环”等近似文本不自动作为跑环别名，以免误伤真实珠结或普通金属连接件。

QY002 的人工反馈构成产品事实修正：原“红色小珠结”应改为红色小珠跑环。该修正写入本次重跑使用的产品分析和 canonical 约束，不通过 PN 特例实现。

### 4.2 结构语义

跑环约束必须表达以下信息：

- 形态：由多颗小珠串成的独立闭合小环，保持可见的环形轮廓和活动结构。
- 归属：跑环独立于手串主串，不得被并入主串珠序。
- 锚点：保持它在产品图中环绕、套接或连接的主珠、连接件或相邻结构。
- 外观：颜色、珠体形态和肉眼可辨的相对尺寸以产品图为准。
- 不确定边界：产品图无法确认精确珠数或隐藏连接方式时，不推断具体数值或背面结构。

### 4.3 禁止变化

跑环不得被改成：

- 绳结或普通珠结；
- 流苏、链坠或普通垂吊配件；
- 单个金属环、金属片或金属连接扣；
- 一颗普通圆珠或一段主串珠序；
- 环绕、套接或连接到不同对象的装饰。

### 4.4 QC 问题

跑环必须有独立的 `must_keep` 和 `fidelity_checks`。检查问题需要同时验证：

1. 是否仍由多颗小珠形成独立闭合小环；
2. 是否保持原来的环绕、套接或连接对象；
3. 是否没有变成珠结、流苏、金属单环、链坠、普通圆珠或主串的一部分。

任一项不满足时，整体不得 `pass`。跑环缺失、换型或连接对象改变属于核心结构错误，应标记 `must_keep_failed` 和 `core_structure_missing`，整体为 `reject`。

## 5. 本批次定向约束

除跑环规则外，其余错误只修订本次纠错批次的 canonical 约束，不扩充全局关键词表。

| 商品 | 本次必须保持的事实 |

| QY002 | 红色小珠跑环的闭合形态、独立性、环绕或连接对象；黄色圆珠、金色连接环与多色流苏的相对关系 |
| QY004 | 黄色雕刻主件两侧的蓝绿色小珠；叶片提溜位于手链另一侧，不得为了同屏展示而迁移到黄色主件旁 |
| QY013 | 灰棕木纹珠中可见的异形爱心轮廓；不得泛化为普通圆珠、椭圆珠或无明确心形的圆润异形珠 |
| QY020 | 木质薄片珠保持产品图可见的深灰棕色、薄片比例和木纹；透明长珠、桃心主珠、花形连接件及细链吊坠不得重组 |
| QY022 | 橙红大珠、透明切面大珠、黄色方圆主珠、深蓝提溜、镂空金球和灰棕混合珠形的可见邻接顺序不得重排 |
| QY024 | 金色花纹圆珠、黄色透明方形主珠、灰棕方珠和小金色隔珠的可见相邻关系不得改变 |

## 6. 产物与数据流

### 6.1 纠错批次

在 `output/` 下创建独立纠错批次，保留以下内容：

- 7 个商品的产品源图和原产品视角；
- 原先选定的手模参考图副本；
- 修订后的产品分析和 canonical 约束；
- 每个生成任务的完整提示词、模型、提交结果、查询结果和原始生成图；
- 逐张 QC、重跑记录、水印队列和飞书回读验证报告。

历史批次只读，不覆盖原 `generation/01`、`generation/02`、原水印图或历史上传验证记录。

### 6.2 生成任务

按以下清单创建 11 个任务：

- QY002：视角 1、2；
- QY004：视角 1、2；
- QY013：视角 1、2；
- QY020：视角 1、2；
- QY022：视角 1；
- QY024：视角 2；
- QY025：视角 1。

默认使用 `gpt_image_2`、`3:4`、`2K`。每个任务沿用对应旧结果的手模参考图，并按原来主视角顺序提交产品图。双产品视角任务继续明确：内部图 2 是当前成图的主产品视角，内部图 3 只补充同一件手链背侧可见结构；两图不是两件产品或双圈手链。

同一任务首次纠错结果未通过时，只重跑该任务。模型切换遵守现有工作流：累计超过 1 次非 `pass` QC 后，下一次才使用 `nano_banana_v2`。

### 6.3 QC 与交付门槛

每张新图必须同时检查：

- 本商品定向约束全部通过；
- 产品颜色、材质、透明度、纹理、珠形、珠序和配件关系正确；
- 没有新增、删除、复制或重组商品结构；
- 没有迁移产品图中的手腕、手臂、皮肤或背景；
- 手部、接触、遮挡、阴影和佩戴关系自然；
- 没有文字、水印、平台标识或额外首饰。

新生成图只有在完整 `checklist_checks` 和 `fidelity_checks` 均通过后才可进入水印队列。任何失败任务保留原产物和失败 QC，不上传，并仅重跑失败任务。

## 8. 测试策略

代码改动采用测试先行：

1. 先新增失败测试，证明现有跑环约束仍是宽泛的“环状连接结构或活动环”。
2. 测试新规则包含多颗小珠、独立闭合小环、原环绕或连接对象及禁止换型。
3. 测试普通“小珠结”不会自动触发跑环，防止语义过拟合。
4. 测试跑环 `must_keep` 的 QC 问题包含形态、锚点和禁止类型三类检查。
5. 测试提示词完整渲染新跑环约束，并继续通过八层 Prompt contract。
6. 运行相关单测后再运行完整测试集，确认没有影响项链、戒指及其他手串约束。

生成与回传属于真实外部流程，另以产物证据验证：

- 11 个生成任务都有提交、轮询和本地结果；
- 11 张新图都有严格 QC 记录且均为 `pass`；
- 水印脚本成功 11、失败 0；

## 9. 异常处理

- AIReiter 提交响应含错误、积分不足、未接受任务或无输出时，不得记录为成功；按 `aireiter-image-generation` 的兜底规则处理。
- 轮询状态不明确时继续查询已有 `out_task_id`，不得重复提交同一任务。
- 新图 QC 失败时不得水印或上传；保留失败产物并只重跑对应任务。
- 水印失败时只重跑失败队列项，不上传无水印原图。
- 飞书写入前映射不唯一、附件数量不为 2 或找不到目标旧附件时，停止该记录写入并报告，不猜测删除对象。
- 飞书上传后验证失败时，保留已确认的新附件，不删除无法确认归属的附件，并输出待人工处理清单。
