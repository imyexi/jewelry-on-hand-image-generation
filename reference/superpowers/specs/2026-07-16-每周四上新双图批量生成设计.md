# 每周四上新双图批量生成设计

## 1. 目标

读取飞书多维表格“标品库总览表”的“每周四上新”视图，为其中资料已录入的 13 个商品分别生成一张上手图和一张生活场景图。系统按本批浅色画面要求自动选择每个输出角色的最佳参考图并直接生成；只有生成结果通过质量检查并添加 Yuan Studio 水印后，才回填到原记录的“上手图”和“生活场景图”字段。

## 2. 本批范围

本批只处理以下 13 个已有产品图片资料、且两个目标字段当前均为空的商品：

- `SY1443`、`SY1444`、`SY1445`、`SY1446`、`SY1447`、`SY1448`
- `SY1449`、`SY1450`、`SY1451`、`SY1452`
- `SY1454`、`SY1456`、`SY1457`

“每周四上新”视图中其余 56 条尚未录入产品资料的占位记录不下载、不生成、不回填。本批不覆盖任何已有目标字段；如果执行期间某条记录的目标字段被其他人填入，该条记录在写回前停止并等待人工确认。

## 3. 数据源与输出字段

- 商品数据源：用户提供的飞书 Base 链接。
- 数据表：`标品库总览表`。
- 视图：`每周四上新`。
- 商品身份字段：`产品编号`。
- 产品图证据：`产品正面图`、`产品上手/上身图`，必要时辅以 `产品图片` 中的静态图片；视频不作为本批生成输入。
- 产品形态字段：`产品形态`，用于确定手链、戒指或项链等商品形态。
- 产品描述证据：`产品名称`、`产品尺寸&材质说明`。
- 参考图库：项目已配置的线上“AI 生图参考图素材库 / 素材收录池”。
- 回填字段：`上手图`、`生活场景图`。两个字段均为 Base 附件字段，因此将最终通过结果的水印版图片直接作为附件追加到对应记录；同时在本地分别保存无水印原图、水印版图片和完整审计记录。

## 4. 方案

采用用户最终确认的直接生成方案：先按品类建立候选池，再针对每个商品分别排序并自动采用 Rank 1。

1. 根据 `产品形态` 和产品图片确认商品属于手链、戒指或项链；字段值与图片冲突时停止该商品，不自行猜测。
2. 按输出角色分别建立“上手图”和“生活场景图”候选池，不跨角色使用参考图。
3. 先执行本批统一画面门禁，再依据商品品类、珠径、圈数、展示部位和主体可见性进行单品排序。
4. 每个商品、每个输出角色保留 Top 3 作为审计候选，并自动采用通过全部门禁的 Rank 1。
5. 自动决策记录商品编号、输出角色、候选排名、参考素材编号、排序理由、风险和文件校验值。
6. Rank 1 不满足角色、浅色画面或产品适配门禁时不得生成；系统继续检查 Rank 2、Rank 3，三张均不合格则停止该商品对应角色。

## 5. 参考图筛选规则

### 5.1 硬性门禁

候选参考图必须同时满足：

- 图片类型与输出角色一致：上手图只使用手部佩戴或身体佩戴类参考；生活场景图只使用生活场景类参考。
- 画面为浅色背景，整体透亮，优先白色、浅绿色或自然绿色系。
- 光线明亮、干净、高级，不使用明显暗调、黑色、深灰或低照度画面。
- 模特服装为基础款、T 恤或其他素雅松弛服装；避免复杂花纹、强品牌标识、礼服感和高饱和服装。
- 无明显文字、水印、平台标识或会干扰商品展示的元素。
- 佩戴部位、主体空间和构图能够容纳当前商品，不以裁切或遮挡关键结构为代价。

本批浅色门禁是对用户当前任务的明确覆盖，只作用于本批运行，不修改项目现有工作流的默认深色背景规则。

### 5.2 单品排序

通过硬性门禁后，按以下因素综合排序：

- 品类和佩戴部位匹配度。
- 珠径、圈数、链长或戒指体量与参考构图的比例适配度。
- 产品主体正面可见度以及关键配件的展示空间。
- 背景和服装对当前商品颜色的衬托效果。
- 手部、手腕、颈肩或身体姿态的自然程度。
- 参考图原有首饰的移除难度和混入风险。
- 同一商品 Top 3 的构图多样性，避免三个候选仅是近似重复图。

## 6. 自动决策与审计

在 `output/2026-07-16-每周四上新双图/` 下保存候选资源副本、机器可读候选清单和自动决策文件。每个商品的审计记录包含：

- 产品编号、产品名称、品类和尺寸材质说明。
- 产品正面图、产品上手/上身图。
- 上手图 Top 3 和生活场景图 Top 3 的参考素材编号、排序理由、主要风险与文件校验值。
- 每个输出角色最终自动采用的 Rank 和参考素材编号。
- 深色背景门禁已按用户明确授权在本批忽略，浅色背景、角色匹配和产品适配门禁仍生效。
- 图片加载失败或候选不足时的明确阻断原因。

用户于 2026-07-16 明确要求忽略现有技能中的深色背景门禁并直接生成，因此本批无需逐商品参考图确认。该授权只适用于本批，不修改项目技能的默认规则，也不放宽输出角色、产品保真和质量检查要求。

## 7. 生成契约

每个商品建立两个互相隔离的生成任务：`hand_worn` 和 `lifestyle`。每个任务固定使用：

- 图 1：当前输出角色自动选择的最佳合格参考图，只提供人物、姿势、构图、背景、服装、光线和空间关系。
- 图 2：当前商品的产品上手/上身图，作为产品身份的唯一生成输入。
- 产品正面图：只用于产品分析、关键结构确认和生成后质量检查，不作为第三张模型输入。

生成要求：

- 竖版 3:4、2K、真实自然摄影质感。
- 上手图突出真实佩戴关系和产品正面展示。
- 生活场景图保留日常氛围，但产品仍需完整、清晰、可辨识。
- 背景保持浅色、透亮、白或绿色系，画面明亮高级。
- 服装保持基础、素雅、松弛，不添加品牌文字或复杂图案。
- 产品颜色、透明度、纹理、珠径、圈数、排列、主珠、隔珠和配件位置以商品图为准。
- 不继承参考图原有首饰，不新增其他戒指、手链、项链、文字、水印或 logo。

默认使用项目现有 AIReiter `gpt_image_2` 路径。生成任务、提示词、输入副本、任务 ID、返回结果和最终图片全部保存在本批 `output/` 目录，不覆盖历史运行。

## 8. 质量检查与重跑

每张结果必须同时对照产品正面图、产品上手/上身图和自动选择的参考图检查：

- 商品品类、数量、颜色、透明度、纹理、尺寸比例和关键结构正确。
- 圈数、链层、目标指位以及主珠、吊坠、隔珠和配件关系正确。
- 商品佩戴、接触阴影、重力和反光自然，无漂浮、穿模、断裂或融化。
- 人手、手腕、颈肩和身体结构自然，无多指、断指、肢体畸形或明显拼接。
- 构图、背景、服装和光线符合本批浅色要求。
- 未继承参考图首饰，未出现额外商品、文字、水印或 logo。

结果分为 `pass`、`rerun` 和 `reject`。只有 `pass` 可以进入回填。单张结果首次未通过时，依据明确失败项进行一次针对性重跑；第二次仍未通过则停止该商品对应角色的回填，并在最终报告中记录阻断原因，不以低质量结果凑数。

## 9. 水印处理

只有质量检查状态为 `pass` 的最终结果进入水印队列。使用项目已安装的 `yuanyuan-ruyi-watermark` 技能生成独立水印副本，不覆盖无水印原图。

每张水印只包含：

- 第一行：`YUAN STUDIO`。
- 第二行：`PN {产品编号}`，例如 `PN SY1443`。

水印采用底部附近居中的深色半透明圆角徽章，不使用整条底栏，不添加中文品牌名、日期、slogan 或其他文字。逐张记录输入文件、产品编号、输出文件、输入 SHA-256、输出 SHA-256 和处理状态；失败图片不得上传或回填。

## 10. 上传、回填与验证

回填前再次读取 13 条记录，校验：

- `record_id` 与产品编号仍保持原映射。
- 目标字段仍为空。
- 本地无水印结果和水印版结果均存在，质量检查状态为 `pass`，水印处理状态为成功。
- 目标附件字段为空，且待上传水印版结果、生成任务 ID 和输出角色的审计记录一致。

按记录分别写入：

- `上手图`：当前商品通过质量检查并完成水印处理的 `hand_worn` 水印版图片附件。
- `生活场景图`：当前商品通过质量检查并完成水印处理的 `lifestyle` 水印版图片附件。

使用 `lark-cli base +record-upload-attachment` 按记录、字段上传；写入后逐条回读两个字段，并核对产品编号、记录 ID、附件文件名、`file_token`、本地 SHA-256 和成功数量。任一错位、缺失、重复、目标字段被并发修改或回读不一致都视为未完成；已确认正确的其他记录不回滚。

## 11. 失败处理

- 商品资料不完整或无法确定产品结构：该商品停止，保留证据，不猜测补全。
- 参考图库没有满足浅色门禁的候选：记录候选不足，不跨角色或放宽到深色参考图。
- 自动选择的参考图不满足角色或产品适配门禁：继续检查后续 Rank；Top 3 全部失败时不生成。
- 第三方生成失败或超时：保存任务回执，根据现有重试规则处理，不伪造完成结果。
- 质量检查未通过：不回填，并在报告中列出失败项。
- 水印处理失败或水印内容不符合规范：不上传、不回填，保留无水印原图和失败记录。
- 飞书写入或回读失败：停止后续相关写入，保留已完成记录和请求回执，避免重复写入。

## 12. 产物与验收

所有下载、候选、自动决策、生成、质检、水印、上传和回填审计均放在：


最终在 `reference/` 下形成执行报告，逐商品记录：产品编号、记录 ID、两个角色的 Top 3、自动采用的参考图、模型任务 ID、质量检查状态、无水印文件、水印版文件、水印校验、附件文件名、`file_token` 和回读结果。

本批验收标准：

- 13 个商品每个角色的 Top 3 与自动选择结果完整可追溯。
- 只对通过角色、浅色画面和产品适配门禁的参考图生成。
- 只有质量检查为 `pass` 的结果被回填。
- 所有回填附件均对应已验证的水印版图片，水印精确显示 `YUAN STUDIO` 和 `PN {产品编号}`。
- 不处理 56 条空白占位记录，不覆盖执行期间出现的已有值。
- 最终报告与飞书回读结果一致。
