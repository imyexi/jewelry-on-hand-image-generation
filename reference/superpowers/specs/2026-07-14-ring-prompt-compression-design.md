# 戒指送模 Prompt 精简设计

## 1. 背景

单主石戒与开口戒真实验收表明，当前送模 Prompt 分别为 2988 和 3032 个字符。长度并非来自模型自动扩写，而是公共构建器把产品分析、品类保真、完整 canonical 约束和参考图审计元数据全部展开到同一个 Prompt 中。同一产品结构会在 `visible_appearance`、`special_requirements`、`must_keep`、`forbid` 和 `must_not_change` 中重复出现，导致关键指位与产品身份被稀释。

## 2. 目标与非目标

### 2.1 目标

1. 标准戒指送模 Prompt 控制在 900 至 1200 个字符，超过 1200 字时拒绝送模，不静默截断产品事实。
2. 目标手、目标手指、真实环绕和禁止额外首饰必须位于前 300 个字符内。
3. 单主石戒明确主石、辅石、戒圈的数量、大小和空间关系；开口戒明确两端点、开口和目标中指的解剖位置。
4. 完整产品分析、canonical 约束、参考图排名和匹配理由继续保留在 JSON、审核页和生成元数据中。
5. 保留现有八层 Prompt 契约和动态字段安全边界，不影响手串、项链和带链吊坠。

### 2.2 非目标

1. 本次不压缩手串和项链 Prompt。
2. 不改变 `ProductAnalysis` 或 `ProductFidelityConstraints` schema。
3. 不通过商品编号、素材编号或本次两个样例路径建立特例。
4. 不用字符截断掩盖超长产品分析；超限必须显式失败并要求修订输入。
5. 不在本次调用外部生图接口或替换历史验收产物。

## 3. 方案选择

采用“完整 canonical + 戒指专用精简渲染器”。

未采用直接缩短公共八层模板，因为会同时改变已经稳定的手串和项链行为。未采用删除 canonical 细节，因为这些字段仍是人工确认和逐项 QC 的依据。未采用简单截取前 1200 字，因为关键产品事实可能位于被截断部分。

## 4. Prompt 结构

戒指仍使用以下八层，标题顺序和唯一性不变：

1. `【基础安全边界】`
2. `【两图职责】`
3. `【产品分析与不确定性】`
4. `【品类保真】`
5. `【展示模式】`
6. `【参考构图场景】`
7. `【遮挡与接触物理】`
8. `【禁止项】`

### 4.1 前 300 字优先约束

在基础安全边界之后立即输出戒指最高优先级规则：

- 只生成一枚目标戒指；
- 必须佩戴在确认后的左右手和目标手指根部；
- 戒圈必须真实连续地环绕目标手指；
- 不得换手、换指、跨指、悬浮或贴在手背；
- 禁止生成手镯、手链、第二枚戒指和其他额外首饰。

### 4.2 指位解剖锚点

目标手以手背朝镜头为默认展示。左手拇指位于画面右侧，右手拇指位于画面左侧。目标手指同时用相邻手指定位：中指位于食指与无名指之间，无名指位于中指与小指之间。其他手指使用同样的相邻关系，不依赖“从左数第几根”这种容易受镜像影响的描述。

### 4.3 产品身份压缩

产品身份只输出一次：

- `visible_appearance` 作为主描述；
- 去重后的 `special_requirements` 和 canonical 特定要求作为补充；
- 通用保真规则统一表达颜色、数量、形状、朝向、戒圈粗细、开口和装饰排列不得改变；
- 不再逐项展开 canonical 的 location、visual_shape、relationship、forbid 和 must_not_change。

如果产品可见描述或特定要求包含“开口”，追加“保持现有开口和端点关系，不得闭合或新增开口”。如果包含“主石”，追加“主石数量、形状、颜色、朝向和相对尺寸必须保持；不得新增第二颗主石”。这些规则按产品事实触发，不绑定具体 SKU。

### 4.4 参考图压缩

送模只保留文件名、手部姿势/场景摘要、风格、需要移除的原首饰和镜面规则。路径、rank、score、推荐方式、备注、匹配理由和风险继续保存在审核与生成元数据中，不再发送给模型。

## 5. 长度与错误处理

戒指 Prompt 构建完成后检查字符数：

- `<= 1200`：正常返回；
- `> 1200`：抛出包含实际长度和上限的 `ValueError`，禁止进入生成；
- 不允许静默截断产品外观、特殊要求或不确定边界。

便携校验器同步检查戒指 Prompt 上限、前 300 字优先规则和必要的指位、环绕、禁止额外首饰语义。历史戒指 Prompt 会因超长而不再通过新契约，但历史产物保持只读，不回写。

## 6. 数据流

`ProductAnalysis + ProductFidelityConstraints + ScoredReference` 先经过既有一致性校验。公共入口识别品类为 `ring` 后调用戒指精简渲染器；其他品类继续使用现有公共渲染器。精简渲染器只改变发送给图像模型的文本，不改变分析、审核决策、canonical、QC 和 generation metadata。

## 7. 测试策略

生产修改采用测试先行：

1. 单主石戒 Prompt 不超过 1200 字，前 300 字包含右手无名指、真实环绕和禁止额外首饰。
2. 单主石戒只表达一次产品主体，不展开 reference rank/path/reason 等审计字段。
3. 开口戒 Prompt 不超过 1200 字，明确左手中指位于食指与无名指之间，其他手指不得戴戒指。
4. 开口戒保留珍珠端、燕子端和可见开口，不得闭合。
5. 超长戒指输入显式失败，不发生静默截断。
6. 便携校验器接受新精简戒指 Prompt，拒绝缺少优先约束或超过 1200 字的 Prompt。
7. 既有手串和项链测试保持通过。

## 8. 完成标准

1. 新增测试先按预期失败，再由最小实现修复。
2. 两个真实验收样例重新构建的 Prompt 均不超过 1200 字。
3. 目标手指、真实环绕和禁止额外首饰均位于前 300 字。
4. 完整 Prompt 契约测试与项目完整测试集通过。
5. `reference/prompt-template.md` 和便携技能 Prompt 契约全文修订后与代码一致，不保留“必须发送完整参考审计元数据”等冲突描述。
