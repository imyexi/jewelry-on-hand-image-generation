# 戒指输入质量与失败驱动重试设计

## 1. 背景

单主石戒和开口戒真实验收暴露出三个独立问题：产品结构分析可能缺少清晰细节证据；参考图可能带水印或不是手部微距；重试始终复用 Rank 1 和同一 Prompt，只在累计失败后切换模型。后续真实货盘测试进一步确认：平铺正面图或细节图不适合作为模型的产品身份输入，否则模型需要把平面结构重绘为佩戴结构，容易造成改款。

## 2. 目标

1. `prepare-review` 支持传入已经人工或视觉工具确认的戒指细节图，用于审核、结构分析和 QC 对照；模型产品身份输入固定使用产品上手图。
2. 戒指参考图在现有结构门槛外，拒绝元数据明确标注的水印、logo、平台标识和非手部近景素材。
3. 戒指首次生成使用人工决策 Rank；失败后按 Top 3 中尚未尝试的顺序切换参考图。
4. 根据最新 `qc.json.critical_failures` 注入简短、确定的纠偏指令。
5. 保持现有模型策略：前两次使用 `gpt_image_2`，累计两次非 pass 后第三次使用 `nano_banana_v2`。
6. 每个 generation 目录记录实际 Rank、送模使用的产品上手身份图和纠偏失败码，保证可追溯。

## 3. 非目标

1. 不用固定中心裁切、肤色阈值或黑边阈值自动猜测戒指位置；这类启发式可能裁掉偏心主石或开口端点。
2. 不在核心 Python 包新增 Pillow/OpenCV 依赖。
3. 不用素材编号建立水印黑名单。
4. 不在本次自动修改飞书图片内容或调用 OCR；参考素材表需要把人工/视觉审核发现写入现有备注、场景或取景字段。
5. 不自动覆盖历史 run 和历史 generation 产物。

## 4. 产品身份图

`prepare-review` 新增可选参数 `--product-detail-image`。仅戒指允许使用；文件必须是 jpg/jpeg/png/webp，并复制为当前 run 下的 `input/product-detail.<ext>`。存在细节图时：

- review 包可展示细节图，供人工确认主石、开口端点、戒圈和装饰；
- canonical 可继续记录细节图作为约束分析证据，但不得据此选择模型输入；
- generate 的第二张输入固定使用 `input/product-on-hand.jpg`；
- 每个 generation 目录固定复制产品上手图为 `product-identity.jpg`；
- `product-detail.*` 不进入 AIReiter 的 `--image` 参数，也不作为第三张辅助输入。

没有细节图的历史 run 行为不变。系统不静默生成或替换细节图；细节图只负责提高分析与人工 QC 的可见性，不能替代真实佩戴状态下的产品身份。

## 5. 参考图质量门槛

戒指候选继续要求明确 `ring`、`worn`、目标手指可见、戒面可见度、分指程度、遮挡与裁切风险。新增：

- 备注、场景、风格或原首饰字段明确出现未被否定的“水印”“logo”“平台标识”时拒绝；
- `framing` 有值但不是手部近景、手部特写、手指近景或戒指特写时拒绝；
- `visible_body_regions` 明确包含全身、半身、头发、脸、胸部等宽场景区域时拒绝。

元数据未声明水印不等于视觉确认无水印，因此人工 review 仍必须查看 Top 3；本门槛负责阻止已经被标记的问题素材继续入选。

## 6. 失败驱动重试

每次生成写入 `reference-rank.txt`。读取历史时，优先使用该文件；历史目录没有该文件时，通过 `hand-reference.*` 与 selected reference 的 SHA-256 匹配实际 Rank。

戒指单图决策的重试顺序：首次使用决策 Rank；最新 QC 非 pass 时，选择 Top 3 中尚未尝试的最小 Rank。Top 3 全部尝试后拒绝继续自动重试，要求重新审核素材。

失败码纠偏矩阵：

| 失败码 | 注入纠偏 |
| --- | --- |
| `finger_position_mismatch`、`hand_side_mismatch` | 强调确认手、目标手指相邻解剖关系，其他手指不得戴戒指 |
| `centerpiece_mismatch` | 强调主石数量、形状、颜色、朝向和相对尺寸 |
| `ring_structure_mismatch` | 强调戒面、戒圈、开口端点和装饰排列，不得重设计 |
| `ring_count_mismatch` | 只允许一枚目标戒指，禁止任何额外首饰 |
| `ring_contact_error` | 戒圈必须连续环绕目标手指，背侧真实遮挡，不得悬浮或贴片 |
| `source_hand_leakage`、`source_person_region_migrated` | 产品图只提供戒指身份，不得继承源手、皮肤、衣服或背景 |
| `finger_deformation` | 保持五指解剖正常，不得多指、融指、断指 |

多失败码去重合并为一个 `【本轮纠偏】` 段，并继续受 1200 字戒指 Prompt 上限约束。

## 7. 完成标准

1. 细节图复制、审核展示和 canonical 分析证据有自动化测试；无论细节图是否存在，生成第二张输入和 generation 审计副本都必须是产品上手图。
2. 明确水印和宽场景参考图无法进入戒指 Top 3。
3. 连续三次戒指生成依次使用 Rank 1、2、3，第三次切换 `nano_banana_v2`。
4. 各失败码生成对应纠偏文本，重复规则只出现一次。
5. Top 3 耗尽后安全失败，不重复使用同一参考图。
6. 戒指定向测试和真实样例验证通过；全量测试中的既有无关失败单独报告。
