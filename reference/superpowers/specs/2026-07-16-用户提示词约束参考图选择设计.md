# 用户提示词约束参考图选择设计

**日期：** 2026-07-16  
**状态：** 已确认，待实施  
**适用范围：** `jewelry-on-hand-workflow` 的 `prepare-review` 参考图筛选、排序、审计与后续生成边界

## 1. 目标

移除上手图工作流中固定的深色参考图门禁。每次选择参考图时：

- 用户提供选图提示词时，将其规范化为一组必须全部满足的硬约束；
- 用户未提供选图提示词时，只按适用品类、输出角色以及参考图关键词与当前产品和展示需求的贴合度筛选排序；
- 深色、浅色、户外、自然光等风格都不能成为系统级默认条件；
- 选图提示词只影响参考图选择，不进入最终 AIReiter 生成 Prompt。

产品结构、输出角色、文件有效性、水印风险、参考图适用性和人工 review 等现有安全门禁保持不变。本次变更不允许用选图提示词绕过这些门禁。

## 2. 当前问题

现行行为在三处固定了深色条件：

1. Skill 文档要求三个输出角色通过深色背景 gate；
2. 参考图评分前会过滤掉没有深色信号的候选，并维护特定素材编号例外；
3. 输出角色 Prompt 固定写入“使用深色背景”。

这会让用户即使需要浅色、户外、自然光或其他画面方向，也无法通过正常选图流程表达；同时，选图条件与最终 Prompt 被错误绑定。

## 3. 输入契约

`prepare-review` 新增可选参数：

```text
--reference-selection-prompt "浅色背景；自然光；右手；近景"
```

参数规则：

- 使用中文或英文逗号、中文或英文分号、换行分隔条件；空白项忽略；
- 去除条件首尾空白并按首次出现顺序去重；
- 每个非空条件是一个独立硬约束，不再按空格继续拆分；
- 不做同义词扩写、反义词推断或模型语义改写，避免程序暗中改变用户意图；
- 规范化后没有有效条件时，按“未提供提示词”处理。

示例中的四个条件必须分别得到参考图元数据支持。若用户希望“右手近景”作为一个整体短语匹配，应将其作为单个条件输入；若希望分别匹配，应写成“右手；近景”。

## 4. 候选筛选与排序

### 4.1 基础门禁

无论是否提供选图提示词，候选都必须先通过：

- 文件存在且可读取；
- `图片类型` 与 `output_role` 一致，不跨角色推断或补位；
- `适用品类` 明确包含当前规范产品品类；
- `适用展示模式`、佩戴部位和品类专属结构要求有效；
- 不存在现有规则明确拒绝的水印、平台标识、严重裁切、主体不可见或结构不兼容风险。

基础门禁不包含任何固定的背景明暗、颜色、场景或光线条件，也不保留特定深色素材编号例外。

### 4.2 有选图提示词

基础门禁通过后，将每个规范化条件与以下参考图描述字段组成的白名单语料匹配：

- `style_category`
- `scene_keywords`
- `pose_keywords`
- `recommended_usage`
- `framing`
- `visible_body_regions`
- `product_visibility`
- `hand_visibility`
- `hand_side`
- `hand_orientation`
- `visible_fingers`
- `mirror_relation`
- `collar_type`

匹配采用规范化后的大小写不敏感子串判断。所有条件都命中才保留候选；任一条件未命中即淘汰。`notes`、文件名、素材编号、默认策略和置信度不用于满足用户条件，防止审计性文字或编号造成误命中。

通过硬约束的候选再按现有品类适配、主体展示空间、构图风险和多样性排序。系统不得因候选不足而删除条件、使用同义词替代或退回固定深色策略。

### 4.3 无选图提示词

未提供有效提示词时：

- 只使用适用品类、输出角色和参考图描述关键词与产品分析/展示需求的贴合度进行筛选排序；
- 可以保留安全风险惩罚和 Top 3 多样性控制；
- 不增加深色、浅色、闪光、自然光、服装颜色或其他系统级风格偏好；
- 产品分析本身明确包含的构图或展示需求仍可参与关键词贴合度，但不得被扩展为新的默认风格。

### 4.4 候选不足

每个输出角色仍要求形成 3 张互异的合格候选。少于 3 张时，`prepare-review` 在创建 review 决策前阻断，并报告：

- 基础门禁前后的候选数量；
- 每个选图条件的命中数量；
- 同时满足全部条件的数量；
- 建议用户修改或删除哪些条件，但程序不自动修改。

## 5. 审计与绑定

每个新 run 写入 `analysis/reference_selection_constraints.json`，至少包含：

```json
{
  "schema_version": 1,
  "mode": "prompt_hard_gate",
  "original_prompt": "浅色背景；自然光；右手；近景",
  "normalized_conditions": ["浅色背景", "自然光", "右手", "近景"],
  "matched_fields": ["style_category", "scene_keywords", "pose_keywords"],
  "candidate_counts": {
    "before_base_gates": 0,
    "after_base_gates": 0,
    "after_prompt_gates": 0
  },
  "condition_match_counts": {},
  "candidate_evaluations": []
}
```

无提示词时仍写该文件，`mode` 为 `keyword_relevance_only`，原始提示词为空，条件列表为空。每个候选评估记录基础门禁结论、每项条件命中字段、淘汰原因和最终排名理由。

`selected_references.json` 和 review 决策绑定该约束文件的 SHA-256。约束文件、产品分析、输出角色或参考源快照发生变化时，旧 Top 3 和旧决策失效，必须重新执行 `prepare-review`。

## 6. 与生成 Prompt 的隔离

选图提示词是选择阶段输入，不是生成阶段输入：

- `build_generation_prompt`、戒指短 Prompt 和 AIReiter `--prompt` 都不得接收或拼接 `original_prompt`、`normalized_conditions`；
- `prompt.txt` 中不得出现仅由选图提示词带入的文本；
- 生成阶段仍使用已选参考图作为内部图 1，并可读取该参考图自身的风格、场景、姿势等元数据；
- 输出角色公共指令移除固定“使用深色背景”，改为中性的角色、产品清晰度和禁止文字要求；
- 若某批任务另有独立、明确确认的生成画面要求，应通过生成需求入口处理，不能从选图提示词自动复制。

因此，选图提示词与参考图元数据可能表达相似内容，但两者来源必须可区分：最终 Prompt 可以描述已选参考图本身，不得宣称这些描述来自用户的选图提示词。

## 7. 兼容与迁移

- 新建 run 使用本设计；没有 `reference_selection_constraints.json` 的历史 run 保持只读，不自动补写；
- 历史 run 不得在新版本中直接继续 `record-decision` 或 `generate`，需要新建 run 并重新 `prepare-review`；
- 删除固定深色文本 gate、特定深色素材编号例外及对应错误文案；
- 现有品类安全 gate、角色分类、参考图 Top 3、人工 review、产品保真与 QC 契约不放宽；
- `jewelry-product-hero-workflow` 是独立 Skill，本次不覆盖用户对该文件的并行修改。

## 8. 文档修订范围

实施时必须整体修订以下内容，不能只在末尾追加例外：

- `skills/jewelry-on-hand-workflow/SKILL.md`
- `skills/jewelry-on-hand-workflow/references/workflow.md`
- `skills/jewelry-on-hand-workflow/references/prompt-contract.md`
- `reference/manual-workflow.md`
- `reference/prompt-template.md`

全文删除或改写“所有角色默认深色”“深色背景硬 gate”“固定深色例外编号”等陈述，并明确选图提示词与生成 Prompt 的隔离边界。

## 9. 测试设计

实施前先建立会失败的基线测试，至少覆盖：

1. 无提示词时，浅色和深色候选都不会仅因明暗被排除；排序只受适用品类、角色、关键词贴合度和安全风险影响。
2. 有提示词时，候选必须满足全部条件；只满足部分条件不能进入 Top 3。
3. 条件不足 3 张时，错误信息包含逐条件命中数且不自动放宽。
4. 选图约束文件完整保存原始输入、规范化条件、候选命中证据和 SHA-256 绑定。
5. 选图提示词变化会使旧 Top 3 和旧决策失效。
6. 最终 `prompt.txt` 不包含选图提示词原文，也不含固定“使用深色背景”。
7. 已选参考图自身的场景和姿势元数据仍能进入现有参考构图区块。
8. CLI 帮助、Skill 便携性检查和相关文档不再宣称默认深色。

测试过程、日志和临时 run 全部写入 `output/`。

## 10. 验收标准

- 用户能在每次 `prepare-review` 时选择是否提供结构化选图提示词；
- 提供提示词时执行全条件硬匹配，不匹配候选被明确淘汰；
- 未提供提示词时没有任何背景明暗或风格默认值；
- 每次选图都有可追溯的约束、命中证据、候选数量和绑定摘要；
- 候选不足时阻断且不静默降级；
- 选图提示词不进入 AIReiter Prompt；
- 现有产品保真、安全、角色和人工 review gate 保持有效；
- Skill、参考文档、实现与测试对上述规则表述一致。
