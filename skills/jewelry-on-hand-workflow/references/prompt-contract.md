# 参考底图替换 Prompt Contract

## 输入职责

生成 Prompt 必须明确：以参考底图为底图进行编辑，不是围绕参考图重新设计或重新生成场景。

参考底图是人物、姿势、手势、构图、景别、服装、背景、光线、留白和替换位置的唯一来源。内部图 1 对应 `scene-reference.*`；它锁定人物身份、身体区域、手侧和动作、镜头与裁切、主体位置、服装、发型、背景道具、光线色调、留白和目标首饰位置。

产品上手图只提供目标珠宝身份。内部图 2 对应 `product-reference.*`；它只提供品类、数量、珠子/链条/戒圈/主石/吊坠/金属件、排列、连接、方向、颜色、材质、透明度、纹理、反光和肉眼可见尺寸。内部图 2 的人物、皮肤、手腕、手臂、手指、颈部、胸部、服装、头发、脸和背景一律不得迁移。

## 固定开头

Prompt 的第一段必须同时表达下列正向编辑契约：

```text
这是参考底图编辑任务。以参考底图为底图进行编辑，保持未授权区域的画面结构。
只允许移除参考图原首饰并在同一位置换入一件目标产品，
以及重建必要接触阴影、局部遮挡和受力关系；可清除小面积水印或平台标识。
```

不得在此段之前放置产品风格、输出规格、摄影术语或品类构图要求。

## 快照优先

构图字段只能来自已确认 `reference-composition-snapshot.json`，并按以下顺序进入 Prompt：

1. `output_role`、`framing`、`camera_angle`、`subject_placement`；
2. `visible_body_regions` 和 `pose`；
3. `clothing`、`background`、`lighting`；
4. `replacement_target`、`other_jewelry_to_remove`、`text_or_ui_risk`。

产品 analysis 与 canonical 只能补充珠宝身份和佩戴物理。只要产品字段、角色模板或自由文本与确认快照的景别、姿势、手侧、服装、背景、光线、留白或替换位置冲突，就停止构建并要求重新 `prepare-review`；不得选择性忽略快照、合并成折中构图或在 generation 阶段修正快照。

## 允许修改区域

允许修改仅限：

1. 参考底图全部原首饰及其直接接触阴影；
2. 快照确认的唯一替换位置；
3. 新产品与人体/衣物接触处的必要遮挡、受力和小范围融合；
4. 小面积水印、logo 或平台标识覆盖区域。

不得重新生成人物或环境，不得换脸、换肤色、换姿势、换手势、换衣服、换景、改变人物位置、改变镜头、裁切、推进、放大或把 `lifestyle` 改成产品特写。展示面积不足不是生成阶段的裁切理由，应回到参考候选选择。

## 固定层序

Prompt 采用以下层序，不允许品类模板越权覆盖前层：

1. 底图编辑声明；
2. 双图职责；
3. 已确认参考构图快照；
4. 唯一允许修改区域；
5. 产品 canonical 与 `must_keep`；
6. 品类佩戴物理；
7. 角色保留要求；
8. 通用禁止项与输出质量。

## 产品保真

- 产品身份、数量、结构、排列、连接、方向、颜色、材质、透明度、纹理、反光和可见比例以内部图 2 与 canonical 为准。
- 产品分析五字段必须完整；项链使用 `schema_version=2` 与 `pendant_semantics`。普通项链保持 `absent/0/null/forbid`，带链吊坠保持 `present/1/实际层/forbid`；不得自动补链或推断不可见扣头、背面和连接。
- 产品图中的人体和背景永远不是 identity 内容。细节图只用于 review、结构分析和 QC，不进入 Prompt 的模型图片列表。

## 品类规则

### 手串/手链

保留珠序、主珠、配珠、隔圈、金属件、透明度、纹理和尺寸感；按照快照位置自然环绕手腕。禁止复制产品、改变珠序或迁移产品图手腕。

### 项链与带链吊坠

保留同一产品 1 至 3 层、长度等级、层间落差、主吊坠数量/所属层/方向/连接关系。`worn` 只按快照指定颈胸位置替换；`hand_held` 只按快照指定手指接触与重力垂落替换。品类规则不得注入“锁骨近景”“半身”等快照外构图。

### 戒指

内部图2是戒指身份唯一来源。固定 `ring_count=1`，以 `hand_side`、`finger_position`、`ring_wear_style=finger_base` 约束目标位置；戒指身份唯一来源只决定戒圈、戒面/主石、镶嵌、颜色和材质。最高优先级：只生成一枚目标戒指；禁止生成手镯、手链、第二枚戒指。按快照保持手背/掌侧朝向和手指姿势，让戒圈自然环绕手指；不可见戒圈背面不得猜测。

## 角色规则

- `hand_worn`：保持快照中的手侧、掌向、手臂角度、手指关系、目标身体部位和景别。项链可按已确认 `hand_held` 策略手持，但不能借此改变构图。
- `lifestyle`：保持原半身、全身或环境景别、人物位置、服装、道具和生活环境。产品在画面中较小也不得改成局部特写。
- `hero`：当前 Skill 不构建 Prompt，必须交给独立主图 Skill。

## 长度与校验

最终 Prompt 不超过 1200 字。优先删去重复修饰词，不得删掉底图锁定、允许修改区域、快照字段、产品 canonical 或冲突停止语义。

`scripts/validate_prompt_contract.py` 以四输入校验：Prompt、确认快照、`product-analysis.json`、`product-fidelity-constraints.json`。必须拒绝职责反转、无快照构图指令、角色冲突、超预算、宽泛重绘、产品图人体迁移和编码损坏。UTF-8 解码失败或任一摘要不一致时，在调用 provider 前返回非零退出码。
