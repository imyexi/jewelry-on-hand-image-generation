# QC Checklist

QC 只能使用 `pass`、`rerun` 或 `reject`。

## Pass 要求

全部满足才可以 `pass`：

- 产品身份可信，主珠、配珠、隔圈、金属件、颜色和比例没有明显漂移。
- 没有复制参考图里的旧首饰。
- 没有新增戒指、指环或额外首饰，除非目标产品包含它们。
- 没有文字、水印、logo 或平台标识。
- 手指、手掌、手腕、皮肤纹理自然。
- 手串贴合目标手腕，遮挡和接触阴影合理。
- 手腕宽度、手臂轮廓、肤色、皮肤纹理与参考手连续。
- 没有把产品原图里的粗手腕、局部手臂或皮肤块随手串迁移到新图。

## Rerun 场景

以下情况通常使用 `rerun`：

- 产品主体基本正确，但轻微变小、偏色、局部裁切或细节不足。
- 手部轻微不自然，但没有严重畸形。
- 首饰贴合关系略假，接触阴影需要加强。
- 背景和风格可用，但不够理想。
- 存在可通过换参考图、强化 prompt 或继续使用默认 `gpt_image_2` 修复的问题。

## Reject 场景

以下情况应 `reject`：

- 产品被重设计，核心配件缺失或变成另一款。
- 参考图首饰、戒指或额外首饰被复制进成图。
- 出现文字、水印、logo。
- 手部明显畸形，多指、断指、融指或关节错位。
- 手串和产品原图手腕作为整体贴片迁移。
- 粗手腕贴到细手上，肤色或轮廓出现明显断层。

## 原图手腕/手臂迁移检查

必须逐项检查并在 `qc.json` 中写明：

- 手串周围是否出现和参考手不一致的皮肤块。
- 手腕上下宽度是否突变。
- 产品原图里的粗手腕、局部手臂、掌纹、指甲是否出现在成图中。
- 手串是否和原图手腕作为一个贴片块移动到参考图。
- 参考手的肤色、光线和皮肤纹理是否连续。

没有这项检查，不能判定为 `pass`。

## QC 与模型兜底联动

QC 结果会影响下一次生成模型：

- `pass` 不计入失败次数。
- `rerun` 和 `reject` 都计入 QC 未通过次数。
- 同一 run 内 QC 未通过次数为 0 或 1 时，下一次仍使用默认 `gpt_image_2`。
- 同一 run 内 QC 未通过次数超过 1 次后，下一次才使用 `nano_banana_v2` 兜底。
- 写 QC 前不要手动删除或绕过旧 `generation/NN/`；缺少 `qc.json` 的非空目录必须先处理。

## QC 记录格式

```json
{
  "status": "pass",
  "passed": ["产品保真通过", "原图手腕/手臂迁移检查通过"],
  "failed": [],
  "notes": "未发现源图手臂局部贴片、粗手腕或皮肤块迁移。"
}
```
