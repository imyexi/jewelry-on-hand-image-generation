# Troubleshooting

## 额外戒指或参考图首饰

原因：

- 参考图中本来有戒指、指甲装饰或其他首饰。
- `ignored_reference_jewelry` 为空或未写入 prompt。
- prompt 只要求“产品保真”，没有禁止参考图首饰。

处理：

- 换更干净的 rank。
- 在 prompt 中明确 `不要把内部图1里的原有首饰迁移到新图`。
- 补充 `No rings, no finger rings, no extra jewelry`。

## 原图手腕或手臂随产品迁移

原因：

- 内部图2是完整产品上手图，模型把手串和原手腕当成同一个视觉主体。
- prompt 过度强调“尺寸感”“贴合关系”，但没有切断皮肤/手腕来源。
- QC 只看手部自然，没有看来源一致性。

处理：

- 确认 prompt 包含产品隔离句。
- 确认 prompt 包含“手腕宽度、手臂轮廓、皮肤连续性和肤色必须以内部图1为准”。
- 如果重复失败，使用窄手串 crop 或更清晰产品图。
- 默认和首次重跑仍使用 `gpt_image_2`；同一 run 内 QC 未通过超过 1 次后，下一次才使用 `nano_banana_v2` 兜底。
- QC 必须写明原图手腕/手臂迁移检查。

## 手部不和谐

原因：

- 参考图手部姿势复杂。
- 模型对手腕接触、手指比例或皮肤连续性控制不足。
- 参考图和产品图手腕粗细差距大。

处理：

- 换手腕露出更完整、背景更简单的参考图。
- 默认使用 `gpt_image_2`；不要在 0 或 1 次 QC 未通过时直接切 `nano_banana_v2`。
- 强化内部图1作为手部唯一来源。
- 如果手部畸形明显，直接 `reject`，不要只靠后期解释。

## 何时使用 nanobanana 兜底

不要把 nanobanana 当作默认补救。模型选择规则固定为：

- 首次生成：`gpt_image_2`。
- 同一 run 内只有 1 次 QC 未通过：继续 `gpt_image_2`，先改参考图或强化 prompt。
- 同一 run 内 QC 未通过次数超过 1 次：下一次生成改用 `nano_banana_v2` 兜底。
- 兜底前必须保证每个历史 `generation/NN/` 都有 `qc.json`；否则先补 QC，不能跳过。
- 兜底生成后仍必须按 QC checklist 检查原图手腕/手臂迁移。

## 产品漂移

原因：

- 动态字段只写材质名，缺少肉眼可见细节。
- prompt 把产品“美化”成更常见款式。
- 参考图中已有相似首饰，模型混淆。

处理：

- 具体描述珠子颜色、透明度、隔圈、金属件、排列。
- 保留固定产品保真句。
- 明确产品只以内部图2为准，参考图只提供手部与构图。

## Prompt 编码损坏

原因：

- PowerShell heredoc、管道或默认编码把中文写坏。
- prompt 出现 `???`、`锟`、`�`。

处理：

- 用 Python UTF-8 写入中文文件。
- 提交前运行 `scripts/validate_prompt_contract.py`。
- 发现乱码时不要提交 AIReiter。

## AIReiter 轮询失败

原因：

- 网络或 SSL 瞬时失败。
- 已提交任务，但等待结果失败。

处理：

- 先读取 `submit.json`。
- 使用原 `out_task_id` 继续查询。
- 不要在无法确认失败时重复提交，避免同一 SKU 产生重复任务。
